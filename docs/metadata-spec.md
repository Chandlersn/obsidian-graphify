# 元数据规范 v2.2（简化版）

## 设计原则

1. **简洁优先** - YAML 仅 4 行，避免冗余
2. **正文为王** - 内容直接呈现，无需额外区块包裹
3. **概念索引** - 关键概念独立区块，便于快速定位
4. **Wiki 链接** - 相关笔记集中管理，图谱联动

---

## 简化模板

```markdown
---
type: original
created: 2026-04-19
tags: ["标签1", "标签2"]
---

# 笔记标题

> 一句话核心观点。

正文段落...

---

正文段落（段落间用 --- 分隔）...

（正文最后一句不加粗）

## 关键概念

- **概念名** → 简要解释
- **概念名** → 简要解释

---

## 相关笔记

- [[笔记名]] (关联理由)

---
```

---

## YAML 字段详解

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | enum | ✅ | 内容类型：`original` / `collected` / `extracted` |
| `created` | date | ✅ | 创建日期：`YYYY-MM-DD` |
| `tags` | array | ✅ | 标签：最多 5 个 |

### type 类型说明

| 值 | 含义 | 存放位置 |
|----|------|----------|
| `original` | 用户原创 | `02-Knowledge/` 或 `03-Projects/` |
| `collected` | 收藏原文 | `04-Materials/Articles/` |
| `extracted` | 提取要点 | `02-Knowledge/` |

---

## 正文格式规范

### 段落分隔

- 段落之间使用 `---` 分隔（保持原文风格）
- 正文最后一句与关键概念之间无需分隔符

### 禁止项

- 正文最后一句不加粗
- 不重复 YAML 内容（如 core_theme、keywords）
- 不添加"正文内容"包裹区块

---

## 关键概念格式

```markdown
## 关键概念

- **概念名** → 简要解释
- **概念名** → 简要解释
```

### 提取规则

1. 从正文提取高频关键词
2. 在概念后附带简要解释（50 字内）
3. 最多 8 个概念

---

## 相关笔记格式

```markdown
## 相关笔记

- [[笔记名]] (关联理由)
- [[笔记名]] (关联理由)

---
```

### 链接规则

1. 由 `wiki_link_builder.py` 自动建立
2. 关联理由用括号标注（匹配关键词）
3. 最多 8 个链接

---

## 分类映射

```
category 判断规则：
- 涉及技术/AI/开发 → Knowledge
- 项目开发文档 → Projects
- 外来文章收藏 → Materials
- 系统配置/设计 → System

subcategory 判断规则：
- 提到「智慧」「认知」「哲学」→ Knowledge/Wisdom
- 提到「LLM」「Agent」「模型」→ Knowledge/AI-ML
- 提到「架构」「设计」「分布式」→ Knowledge/Architecture
- 提到具体项目名 → Projects/{项目名}
```

---

## Agent 收录流程

```
1. 识别来源（source）
   - user → 直接录入
   - wechat/web → 询问处理方式

2. 分析内容 → 判断 category + subcategory

3. 应用简化模板 → 保存文件

4. 建立 Wiki 链接 → wiki_link_builder.py

5. 更新图谱 → update_graph_html.py

6. 打开浏览器 → 展示最新图谱
```

## 查询用途

```python
# 查询所有原创
filter: type=original

# 查询特定日期
filter: created=2026-04-19

# 查询包含特定标签
filter: tags CONTAINS "智慧"

# 查询 Wisdom 分类
path: 02-Knowledge/Wisdom/
```

## 维护规范

1. **created 创建后不变** - 记录首次录入时间
2. **tags 最多 5 个** - 精简标签数量
3. **关键概念与正文关键词一致** - Agent 自动同步
4. **相关笔记每次更新图谱时刷新** - 自动重建链接
